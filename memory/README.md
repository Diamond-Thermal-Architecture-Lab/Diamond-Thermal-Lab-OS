# Engineering Memory

The memory layer stores reusable, public-safe engineering memory for Diamond Thermal Lab OS. It supports early screening, pattern comparison, validation planning, supplier review, and claim safety.

Memory files are not a source of final customer claims. Screening ranges and qualitative entries must be checked against supplier data, literature, measurements, or reviewed case evidence before customer-facing use.

## Files

- `material_properties.yml`: conservative screening ranges and placeholders.
- `tbr_ranges.yml`: qualitative interface thermal boundary resistance memory.
- `failure_modes.yml`: common reasoning and integration failure modes.
- `architecture_lessons.yml`: reusable decision lessons.
- `validation_methods.yml`: validation method memory.
- `supplier_observations.yml`: supplier-data review observations.
- `simulation_residuals.yml`: simulation uncertainty memory.
- `validated_assumptions.yml`: reviewed assumptions and their evidence status.
