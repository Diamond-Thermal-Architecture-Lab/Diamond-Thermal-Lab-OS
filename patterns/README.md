# Pattern Library

The pattern library gives Diamond Thermal Lab OS reusable thermal architecture patterns for early decision work. Patterns are not recommendations by themselves. They help engineers compare routes, identify assumptions, request supplier data, and keep claims tied to validation.

Use pattern IDs in architecture genomes, scorecards, red flags, validation plans, supplier requests, customer memos, and claim ledgers when a case uses a reusable decision pattern.

Every pattern is public-safe by default and avoids customer-specific data, proprietary process recipes, measured performance claims, and automatic diamond recommendations.

See `pattern_index.yml` for route type, maturity level, primary risks, required validation, and related case artifacts.
